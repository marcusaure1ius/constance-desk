# Constance — Персональная канбан-доска

Спецификация дизайна персонального канбан-трекера задач с деплоем на Vercel.

## Цель

Помочь одному пользователю быстро создавать задачи, отслеживать их статус через настраиваемые колонки и формировать еженедельные отчёты о проделанной работе.

## Технологический стек

| Слой | Технология |
|------|-----------|
| Фреймворк | Next.js 16, App Router |
| UI | shadcn/ui + Tailwind CSS 4 |
| Drag & Drop | `@hello-pangea/dnd` |
| ORM | Drizzle ORM |
| База данных | Neon Postgres (Vercel Marketplace) |
| Авторизация | 6-значный PIN, bcrypt-хеш, сессия через HTTP-only cookie |
| Деплой | Vercel |
| На будущее | Telegram-бот (отдельные API Routes) |

## Server Actions

| Action | Назначение |
|--------|-----------|
| `createTask` | Создание задачи с дефолтами (start_date=сегодня, position=последняя) |
| `updateTask` | Редактирование любых полей задачи |
| `moveTask` | Перемещение между колонками, пересчёт позиций, установка/сброс completed_at |
| `deleteTask` | Удаление задачи |
| `reorderColumns` | Изменение порядка колонок |
| `createColumn` | Добавление новой колонки |
| `updateColumn` | Переименование колонки |
| `deleteColumn` | Удаление колонки (заблокировано, если в ней есть задачи — сначала нужно их перенести) |
| `createCategory` | Добавление категории в справочник |
| `updateCategory` | Переименование/смена цвета категории |
| `deleteCategory` | Удаление категории |

## API Routes (для будущего Telegram-бота)

- `POST /api/tasks` — создать задачу
- `GET /api/tasks` — список задач (с фильтрами: колонка, категория, приоритет, planned_date)
- `PATCH /api/tasks/:id` — обновить задачу
- `GET /api/report?week=2026-W14` — отчёт за неделю

Server Actions и API Routes используют общий сервисный слой в `lib/services/` — без дублирования логики.

Авторизация API: заголовок `X-API-Key` с сырым PIN, проверяемый через bcrypt-хеш в БД.

## Дизайн интерфейса

### Десктоп — Канбан-доска

Шапка с названием «Constance», кнопки: «План на сегодня», «Отчёт», «Настройки».

Под шапкой: кнопка «+ Добавить задачу».

Основная область: горизонтальные колонки рядом друг с другом. У каждой колонки заголовок со счётчиком задач. Задачи — перетаскиваемые карточки.

**Карточка задачи** отображает:
- Цвет приоритета как левая полоска (красный / жёлтый / серый)
- Название задачи
- Категория в виде бейджа
- Плановая дата (или дата завершения с галочкой в последней колонке)

Клик по карточке — открывает её для редактирования.

### Мобильная версия — Табы

Шапка компактная, кнопки в виде иконок.

Табы под шапкой: по одному табу на каждую колонку, активный подчёркнут.

Кнопка «+ Добавить задачу» под табами.

Вертикальный список карточек задач в активном табе.

Перемещение задач на мобильном: свайп по карточке или кнопки «←» «→» в развёрнутом виде карточки.

### Модалка «Создать задачу»

Поля:
1. Название (текстовое поле, обязательное)
2. Описание (textarea, опциональное)
3. Дата начала (date picker, по умолчанию: сегодня)
4. Плановая дата (date picker, опциональное)
5. Категория (выбор из справочника)
6. Приоритет (три toggle-кнопки: красный/жёлтый/серый, по умолчанию: серый)
7. Чекбокс «Создать ещё одну» — при включении форма очищается, но модалка остаётся открытой

Основная кнопка: «Создать задачу».

### Сайдбар «Отчёт»

Открывается справа.

Два инпута сверху:
1. «Текущая неделя» — показывает текущую неделю по умолчанию
2. Выбор недели — навигация к произвольной неделе

Отображает:
- Диапазон дат недели (например, «30 марта — 5 апреля 2026»)
- Сводка: «Выполнено: 5 из 8 задач» с прогресс-баром
- Список выполненных задач: название, бейдж категории, описание, дата завершения

Кнопка «Копировать текст» внизу — форматирует отчёт как текст:
```
Отчёт за неделю: 30.03 — 05.04.2026
Выполнено: 5 из 8 задач

✓ Название задачи (Категория) — 01.04.2026
  Описание задачи
```

### Модалка «План на сегодня»

Показывает все задачи, у которых `planned_date = сегодня`, сгруппированные по текущей колонке.

Счётчик задач внизу.

## Ключевые поведения

### Drag & Drop (оптимистичные обновления)
- Пользователь перетаскивает карточку → UI обновляется мгновенно → Server Action отправляется в фоне
- При ошибке сервера → карточка возвращается на место с тостом об ошибке

### Автоматическое заполнение `completed_at`
- Задача попадает в последнюю колонку (наивысший `position` среди колонок) → `completed_at = now()`
- Задача возвращается из последней колонки → `completed_at = null`

### Авторизация по PIN
- Первый визит (PIN не задан) → экран «Придумайте PIN»
- Последующие визиты → экран ввода PIN через InputOTP (shadcn)
- Сессия: HTTP-only cookie, срок жизни 30 дней
- API Routes проверяют заголовок `X-API-Key` для доступа бота

### Страница настроек
- **Колонки**: добавить, переименовать, удалить (заблокировано, если в колонке есть задачи — сначала нужно их перенести), перетащить для изменения порядка
- **Категории**: добавить, переименовать, сменить цвет, удалить
- **PIN**: смена PIN (требует подтверждение текущего PIN)

### Запрос еженедельного отчёта
```sql
-- Задачи, выполненные на этой неделе
SELECT * FROM tasks
WHERE completed_at >= начало_недели AND completed_at < конец_недели;

-- Задачи, начатые на этой неделе (для сводки «X из Y»)
SELECT COUNT(*) FROM tasks
WHERE start_date >= начало_недели AND start_date < конец_недели;
```

«X из Y» означает: X выполнено на этой неделе, Y начато на этой неделе. Задачи, перенесённые с прошлых недель и выполненные на этой, учитываются в X, но не в Y.
